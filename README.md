# Banjara — Full House build

The Full-House tier from the THEKULTHOUSE restaurant catalogue, implemented end to end:
custom website + digital menu + reservations + pre-orders (pickup & with-table) +
mock-UPI prepayment + owner admin panel. Static HTML/CSS on Vercel, Supabase for
data + auth, Vercel serverless functions for payment & email.

## Stack

| Layer | Tool | Why |
|---|---|---|
| Hosting | Vercel | already in use; static + serverless on one platform |
| Database & Auth | Supabase (Mumbai region) | postgres + RLS + free tier covers a single restaurant |
| Client | Vanilla JS (ESM, no bundler) | preserves the existing HTML design; zero build step |
| Payments | Mock now → Razorpay later | UPI/cards via hosted checkout; signature verified server-side |
| Email | Resend (planned) | confirmations on reservation + paid order |

## Project layout

```
/                    static HTML pages (index, menu, about, gallery, reservation, contact, pre-order, admin)
/js/                 ESM modules — supabase client, menu fetch, cart, admin
/api/                Vercel serverless functions — payments, email
/supabase/migrations one .sql per migration, in apply order
/supabase/seed       seed data (menu)
/scripts/            one-off data tools (menu extractor, etc)
```

## Database

Two migrations applied, in order:
1. `0001_init_schema.sql` — tables, types, indexes, updated_at triggers
2. `0002_rls_policies.sql` — row-level security: public read menu / write pending reservations + placed orders; owner authenticated for everything else

Seed: `supabase/seed/menu_seed.sql` (regenerated by `python3 scripts/extract_menu.py`).

## Public config

`js/config.js` holds the Supabase URL + publishable anon key. Both are **safe** in
the client — RLS is what protects writes. Service role key and Razorpay secret
never leave Vercel env vars.

## Payment mode flag

`PAYMENT_MODE` in `js/config.js`:
- `"mock"` — current. Table pre-orders mark `payment_status = 'mock_paid'` after a
  confirmation step. Visibly labelled "DEMO MODE" so the client never thinks it
  took real money.
- `"razorpay"` — once Razorpay keys are in Vercel env. No client-side code change
  beyond this flag; the serverless verify function picks the right path.

## Admin

`admin.html` behind Supabase Auth. Email/password login. CRUD on menu items +
categories; live list of reservations and orders with status controls.

## Owner login at handover

Placeholder `owner@banjara.local` documented in `js/config.js`. Real credentials
are provisioned by the client; we never store live customer creds in the repo.
